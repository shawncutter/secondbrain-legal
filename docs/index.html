Docker Containers

Module: github.com/p2l/secondbrain · Go 1.25 · Binary: secondbrain (Cobra subcommands)

Pattern: All 26 agents run as goroutines in a single unified Docker container (go-agents), each on its own port. Gateway reverse-proxies all traffic through port 8022.

Database Infrastructure

PostgreSQL :5437

Primary relational database. Each agent has its own schema/tables. Encryption at rest. Multi-tenant isolation.

Redis :6380

Cache layer, event bus (pub/sub), session storage. 256MB limit. In-memory fallback if unavailable.

Neo4j :7475/:7688

Knowledge graph. Stores entity relationships, document connections, wikilinks from Obsidian vault.

MSSQL :1434

Legacy integrations. Microsoft 365 and enterprise data sources.

Database Schemas (Key Tables)

Social Agent

metrics_snapshots — Social media metrics over time
task_snapshots — Tasks from ClickUp/1Medium
task_sync_history — Sync audit log

Tasks Agent

tasks — Unified tasks (external_id, platform, title, status, priority, assignees, tags, due_date)
sync_history — Sync audit log (platform, sync_type, task_count, status)
Indexes on platform, status, assignee, due_date

Calendar Agent

events — Unified calendar events
event_sources — Platform-specific event data
conflicts — Detected scheduling conflicts

Knowledge Agent

file_snapshots — Indexed files (provider, file_id, account, path, processing_status)
document_content — Extracted text (file_id, text_content, word_count)
entities — Named entities (type, name, normalized_name, confidence)
topics — Extracted topics (name, keywords, parent_topic_id)

Journal Agent (Day One)

entries — Day One entries with full metadata
tags, entry_tags — Tag relationships
locations, weather, photos, activity_types

Event Bus

All agents communicate via Redis pub/sub through internal/shared/messaging/. Falls back to in-memory if Redis unavailable (note: cross-container events are lost in fallback mode).

Pattern: Publish events on state changes → Subscribers react asynchronously → Orchestrator coordinates multi-agent workflows.

Middleware Stack

Recovery → RequestLogger → CORS → Auth (optional) → RateLimit (optional)

PostgreSQL Retry

// Exponential backoff for PG connections at startup
connectPGWithRetry(ctx, dbURL, maxRetries, agentName)
// Retries: 1s, 2s, 4s, 8s, 16s with SQLite fallback

Credential Encryption

// All OAuth tokens encrypted with Fernet before storage
vault := credentials.NewVault(vaultDir)
vault.Store(platform, accountID, token)  // encrypts
token := vault.Get(platform, accountID)   // decrypts

Quick Start

cd /path/to/SecondBrain

# Build binary
go build -o secondbrain cmd/secondbrain/main.go

# Docker: Start all agents
docker compose --profile agents up -d

# Local dev: infrastructure in Docker, agents directly
docker compose --profile db --profile messaging up -d
./secondbrain gateway serve --port 8022
./secondbrain gmail serve --port 8040

# Run tests
go test ./internal/... ./cmd/... -v

# Help
./secondbrain --help
